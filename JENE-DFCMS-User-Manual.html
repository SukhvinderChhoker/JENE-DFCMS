1.2 Accessing the Application

Open your browser and navigate to:

http://localhost:4200

The login page will appear with the JENE-DFCMS branding — a navy blue background with the system logo and credential fields.

Login Screen

The login page displays the JENE-DFCMS logo at center, two input fields (Email or Username and Password), and a Sign In button in orange. The background uses a navy blue gradient. A "Remember Me" checkbox may appear below the password field.

1.3 Logging In

Use the default administrator credentials or your assigned account:

Username: admin
Password: admin123
  1. Navigate to http://localhost:4200
  2. Enter your username in the Username field
  3. Enter your password in the Password field
  4. Click the Sign In button (orange)
  5. Upon successful authentication, you will be redirected to the Dashboard
⚠ Important
The default credentials (admin / admin123) should be changed immediately after first login for security purposes. Contact your system administrator if you cannot log in.

After logging in, you will see the main application interface with a top navigation bar:

  2. Select Logout from the dropdown menu
  3. You will be redirected to the login page
ℹ Note

Chapter 02

Dashboard

Monitor system activity, view case statistics, and get a real-time overview of operations.

2.1 Overview

The Dashboard is the first screen you see after logging in. It provides a high-level summary of all forensic operations, including case counts, evidence status, and recent activity. The dashboard is role-aware — different users see data relevant to their permissions.

Dashboard Layout

The dashboard displays a row of statistics cards at the top (showing Total Cases, Active Cases, Pending Evidence, Completed Tasks, etc.), followed by charts/graphs showing case distribution by status and investigator workload. Below the charts, a "Recent Activity" feed lists the latest actions across the system.

2.2 Statistics Cards

The top section displays key metrics in colored cards:

CardColorDescription
Total CasesBlueCount of all cases in the system across all statuses
Active CasesGreenCases currently in progress (status: IN_PROGRESS or UNDER_REVIEW)
Pending EvidenceOrangeEvidence items awaiting check-in or review
Completed TasksPurpleTasks marked as completed in the current period
Open IssuesRedOverdue tasks or flagged items requiring attention
Team MembersTealNumber of active users in the system

2.3 Charts and Graphs

Below the statistics cards, the dashboard includes visual analytics:

  • Case Status Distribution: A pie or doughnut chart showing cases by status — NEW, IN_PROGRESS, UNDER_REVIEW, COMPLETED, CLOSED
  • Cases Over Time: A line chart showing case creation and completion trends over the selected period
  • Evidence Status: A bar chart showing how many evidence items are checked in, checked out, or in transit
  • Investigator Workload: A horizontal bar chart showing the number of active cases per investigator

2.4 Recent Activity Feed

The bottom section of the dashboard displays a chronological feed of recent actions:

  • Case created or updated
  • Evidence checked in or checked out
  • Task status changed
  • User roles modified
  • Reports generated

Each activity entry shows the timestamp, user who performed the action, and a brief description. Click on any entry to navigate to the relevant case or evidence item.

💡 Tip
Use the date range picker (if available) to filter dashboard data by a specific time period. This is useful for weekly reviews or monthly reporting.

2.5 Refreshing Data

The dashboard data refreshes automatically every 60 seconds. To manually refresh:

  1. Click the Refresh icon (circular arrow) in the top-right corner of the dashboard
  2. Wait for the loading spinner to complete
  3. All statistics and charts will update with the latest data

Create, update, track, and manage forensic investigation cases through their entire lifecycle.

3.2 Viewing All Cases

Navigate to Cases in the top navigation bar. The Cases list page displays all cases in a searchable, filterable table.

Cases List Screen

The Cases page shows a table with columns: Case ID, Title, Status (color-coded badge), Priority, Assigned Investigator, Created Date, and Last Updated. Above the table, there are filter controls for status, priority, and date range. A search bar allows filtering by case title or ID. A "Create Case" button (orange) appears in the top-right corner.

3.3 Creating a New Case

  1. Click the Create Case button (orange) on the Cases list page
  2. The "New Case" form appears. Fill in the required fields:
    • Title (required): A descriptive name for the case
    • Description: Detailed description of the investigation
    • Priority: Select from LOW, MEDIUM, HIGH, or CRITICAL
    • Assign Investigator: Select an investigator from the dropdown
    • Due Date: Set the expected completion date
  3. Click Save to create the case. The status defaults to NEW
  4. The system creates a unique Case ID (e.g., CASE-2026-001) and redirects you to the case detail page
ℹ Note
Only users with ADMIN, CASE_MANAGER, or INVESTIGATOR roles can create cases. REQUESTER users can submit case requests that require authoriser approval.

3.4 Case Detail Page

Clicking on a case from the list opens the Case Detail page. This page is divided into tabs:

Case Detail Layout

The case detail page has a header showing the Case ID, title, and status badge. Below is a tabbed interface with: Overview (case details and description), Tasks (list of tasks under this case), Evidence (linked evidence items), History (audit trail of all changes), and Linked Cases (related cases). An "Edit" button allows modifying case details, and a "Change Status" dropdown provides status transitions.

3.5 Editing a Case

  1. Open the case detail page by clicking on a case from the list
  2. Click the Edit button in the case header
  3. Modify any editable fields (title, description, priority, due date, assignee)
  4. Click Save to apply changes
  5. The change is logged in the case history with timestamp and user information

3.6 Changing Case Status

Status changes follow the lifecycle defined in section 3.1. Not all transitions are available from every status.

  1. Open the case detail page
  2. Locate the Status dropdown in the case header
  3. Select the new status from the available options (only valid transitions appear)
  4. If transitioning to UNDER_REVIEW, a confirmation dialog appears asking you to confirm the QA handoff
  5. The status change is recorded in the case history with the user's name and timestamp
⚠ Restriction
Once a case is CLOSED, its status cannot be changed. This is a terminal state. Ensure all evidence is properly checked in and all tasks are completed before closing a case.

3.7 Case History

The History tab on the case detail page shows a complete audit trail of all actions performed on the case:

  • Status changes (with from/to values)
  • Field modifications (title, description, priority, assignee)
  • Task additions and updates
  • Evidence links added or removed
  • Case linkage operations

Each history entry displays the timestamp, user, action type, and details of the change. History entries are immutable and cannot be deleted.

3.8 Linking Cases

Some investigations involve related cases. JENE-DFCMS allows you to create explicit links between cases.

  1. Open the case detail page
  2. Navigate to the Linked Cases tab
  3. Click Link Case
  4. Search for the related case by Case ID or title
  5. Select the case and optionally specify the relationship type (e.g., "Related Investigation", "Parent Case", "Duplicate")
  6. Click Confirm to create the link
  7. The linked case now appears in both cases' Linked Cases tabs
💡 Tip
Linking cases is especially useful in complex investigations involving multiple suspects or cross-jurisdictional incidents. Linked cases are cross-referenced in reports.

3.9 Filtering and Searching Cases

Use the filter controls at the top of the Cases list to narrow results:

  • Status filter: Show only cases in a specific status
  • Priority filter: Filter by LOW, MEDIUM, HIGH, or CRITICAL
  • Assigned to: Show cases assigned to a specific investigator
  • Date range: Filter by creation date or due date
  • Search bar: Free-text search across case titles, IDs, and descriptions
⌨ Keyboard Shortcuts
Ctrl + N — Create new case  |  Ctrl + F — Focus search bar  |  Esc — Close modal/dialog
Chapter 04

Task Management

Create, assign, and track tasks associated with forensic cases.

4.1 Tasks Overview

Tasks are individual work items within a case. Each task represents a specific action that must be completed as part of the investigation — such as analyzing a hard drive, reviewing logs, or preparing a report. Tasks help break down complex cases into manageable units and enable progress tracking.

Tasks Tab (within Case Detail)

The Tasks tab shows a table listing all tasks for the current case. Columns include: Task ID, Title, Status, Assigned To, Priority, Created Date, and Due Date. A "Create Task" button appears at the top. Tasks are color-coded by status. Clicking a task opens its detail view.

4.2 Creating a Task

  1. Open the case where you want to add a task
  2. Navigate to the Tasks tab
  3. Click Create Task
  4. Fill in the task details:
    • Title (required): Brief description of the task
    • Description: Detailed instructions or notes
    • Priority: LOW, MEDIUM, HIGH, or CRITICAL
    • Assigned To: Select an investigator or team member
    • Due Date: Expected completion date
  5. Click Save to create the task
  6. The task appears in the Tasks tab with status NEW

4.3 Task Status Tracking

Tasks follow their own status workflow:

NEW
→
IN_PROGRESS
→
COMPLETED

To update a task status:

  1. Open the task detail view by clicking on the task
  2. Use the Status dropdown to change the status
  3. When marking as COMPLETED, you may be prompted to add a completion summary
  4. The status change is logged in the task history
💡 Tip
You can quickly update task status from the Tasks tab by clicking the status badge on each row — it becomes a dropdown when clicked.

4.4 Adding Notes to Tasks

Notes allow team members to communicate progress, share findings, or document issues within a task context.

  1. Open the task detail view
  2. Scroll to the Notes section
  3. Type your note in the text area
  4. Click Add Note to save
  5. The note appears in the chronological notes thread with your name and timestamp
ℹ Note
Notes are visible to all team members assigned to the case. They are included in case export and reports. Be mindful of sensitive information when adding notes.

4.5 Task History

Each task maintains its own audit history, similar to case history. The task history records:

  • Status changes with timestamps
  • Assignment changes (who the task was reassigned to)
  • Priority modifications
  • Notes added by team members
  • Due date changes

4.6 Assigning Investigators and QA

Tasks can be assigned to specific team members for accountability and workload distribution.

  1. Open the task detail view
  2. Click Edit or click directly on the "Assigned To" field
  3. Select the appropriate investigator from the dropdown (shows only users with INVESTIGATOR or QA roles)
  4. Optionally, assign a QA reviewer separately if your workflow requires quality assurance sign-off
  5. Save the changes
⚠ Important
QA reviewers should be different from the investigator assigned to the task to maintain independence. The system may enforce this separation based on your organization's configuration.

4.7 Bulk Task Operations

For efficiency, the system supports bulk operations on tasks:

  • Bulk Status Change: Select multiple tasks using checkboxes and change their status simultaneously
  • Bulk Assignment: Reassign multiple tasks to a different investigator
  • Bulk Delete: Remove multiple tasks (requires admin privileges)
⌨ Keyboard Shortcuts
Ctrl + T — Create new task  |  Ctrl + Shift + A — Select all tasks  |  Delete — Delete selected tasks (with confirmation)
Chapter 05

Evidence Management

Track, manage, and maintain chain of custody for all digital and physical evidence.

5.1 Evidence Overview

Evidence management is a core function of any forensic system. JENE-DFCMS provides comprehensive evidence tracking with check-in/check-out workflows and full chain of custody documentation. Every evidence item is linked to a case and maintains a complete history of handling.

Evidence List Screen

The Evidence page (accessible from the top navigation) displays a table of all evidence items across all cases. Columns include: Evidence ID, Title, Case Reference, Status, Current Holder, Check-in Date, and Last Updated. Filters allow narrowing by case, status, evidence type, and date range. An "Add Evidence" button is available for creating new items.

5.3 Creating Evidence Items

Evidence items are created under a specific case. You can add evidence from the case detail page or directly from the Evidence list.

Via Case Detail Page

  1. Open the case you want to add evidence to
  2. Navigate to the Evidence tab
  3. Click Add Evidence
  4. Fill in the evidence details:
    • Title (required): Descriptive name (e.g., "Suspect's Laptop Hard Drive")
    • Description: Physical description, model, serial number
    • Evidence Type: Select from DIGITAL, PHYSICAL, DOCUMENT, TESTIMONY
    • Hash Value: MD5 or SHA-256 hash for digital evidence integrity
    • Storage Location: Where the evidence is stored
  5. Click Save to register the evidence item

Via Evidence List Page

  1. Navigate to Evidence in the top navigation
  2. Click Add Evidence
  3. Select the associated case from the dropdown
  4. Fill in the evidence details as described above
  5. Click Save

5.4 Check-In / Check-Out Workflow

The check-in/check-out process ensures physical evidence accountability. Every time evidence changes hands, it must be formally checked in or out.

Checking In Evidence

  1. Open the evidence item detail page
  2. Click Check In (available when status is REGISTERED)
  3. Confirm the evidence details and storage location
  4. Optionally add a condition report (notes on physical state)
  5. Click Confirm Check-In
  6. The status changes to CHECKED_IN and a chain of custody entry is created

Checking Out Evidence

  1. Open the evidence item detail page
  2. Click Check Out (available when status is CHECKED_IN or IN_ANALYSIS)
  3. Enter the checkout reason (e.g., "Court presentation", "External lab analysis")
  4. Specify the expected return date
  5. Confirm your identity (system may require re-authentication)
  6. Click Confirm Check-Out
  7. Chain of custody is updated with the checkout record
🔒 Security
Check-out operations require audit trail integrity. All check-out actions are logged with the user's identity, timestamp, and reason. Tampering with evidence records is logged and reported to administrators.

5.5 Chain of Custody

The Chain of Custody tab on the evidence detail page provides a complete, chronological record of every person who has handled the evidence:

Chain of Custody Records

The chain of custody displays a timeline view with entries showing: Date/Time, Action (CHECKED_IN, CHECKED_OUT, TRANSFERRED, RETURNED), Performed By (user name and role), Location, and Notes. Each entry is timestamped and cannot be modified or deleted. An export option allows downloading the chain of custody as a PDF or CSV for court submissions.

Each chain of custody entry includes:

  • Timestamp: Exact date and time of the action
  • Action: Type of custody event
  • Actor: User who performed the action
  • From/To: Custody transfer details
  • Notes: Any additional observations or condition notes
💡 Tip
Regularly export chain of custody records for your cases. These exports are court-admissible and provide an irrefutable record of evidence handling. Use the "Export PDF" button on the Chain of Custody tab.

5.7 Searching and Filtering Evidence

The Evidence list supports powerful filtering:

  • By Case: Show evidence for a specific case

  • By Type: DIGITAL, PHYSICAL, DOCUMENT, TESTIMONY

Chapter 06

User Management

Create accounts, assign roles, and manage user access to the system.

6.1 Overview

User Management is restricted to users with the ADMIN role. From this module, administrators can create new user accounts, edit existing profiles, assign roles, activate/deactivate accounts, and manage system access.

User Management Screen

The Users page (accessible from the top navigation) displays a table of all system users. Columns include: User ID, Full Name, Email, Role (with colored badge), Status (Active/Inactive), Created Date, and Last Login. An "Add User" button appears in the top-right corner. Search and role filter controls are available above the table.

6.2 Creating a New User

  1. Navigate to Users in the top navigation bar
  2. Click Add User (orange button)
  3. Fill in the user details:
    • Full Name (required): User's complete name
    • Email (required): Valid email address for notifications
    • Username (required): Unique login identifier
    • Password (required): Initial password (minimum 8 characters)
    • Role (required): Select the appropriate role from the dropdown
    • Phone: Contact number (optional)
    • Department: Organizational department (optional)
  4. Click Save to create the user account
  5. The user receives a welcome email (if email is configured) with login instructions
⚠ Security
Passwords must be at least 8 characters and include a mix of uppercase, lowercase, numbers, and special characters. The system enforces password complexity requirements.

6.3 Available Roles

JENE-DFCMS uses a role-based access control (RBAC) system. Each user is assigned exactly one role:

RoleBadgeDescription
ADMIN ADMIN Full system access. Manages users, system settings, and all modules. Can perform any operation.
CASE_MANAGER CASE_MANAGER Creates and manages cases, assigns investigators, oversees case progress, manages evidence checkout.
INVESTIGATOR INVESTIGATOR Performs forensic analysis, creates tasks, manages evidence check-in/out, adds notes and findings.
QA QA Reviews completed work for quality and compliance. Can approve or reject tasks and case transitions.
REQUESTER REQUESTER Submits case requests and views assigned cases. Cannot create cases directly; requires authoriser approval.
AUTHORISER AUTHORISER Approves case requests from REQUESTER users. Can view all cases but cannot perform investigations.

6.4 Editing User Details

  1. Navigate to the Users page
  2. Click on the user's name or the Edit button in their row
  3. Modify the desired fields (name, email, role, department, phone)
  4. Click Save to apply changes
ℹ Note
You cannot change your own role to prevent privilege escalation. To change an admin's role, a different admin must perform the action.

6.5 Activating and Deactivating Users

Instead of deleting users (which would break audit trails), administrators can deactivate accounts:

  1. Open the user detail page
  2. Click Deactivate (or Activate for inactive users)
  3. Confirm the action
  4. Deactivated users cannot log in but their historical records remain intact

6.6 Resetting Passwords

  1. Open the user detail page
  2. Click Reset Password
  3. Enter a new temporary password or generate one automatically
  4. The user is notified and must change the password on next login
⌨ Keyboard Shortcuts
Ctrl + Shift + U — Open user management  |  Ctrl + Shift + N — Create new user
Chapter 07

Reports and Analytics

Generate, view, and export comprehensive reports on cases, evidence, and team activity.

7.1 Reports Overview

JENE-DFCMS includes a reporting module that provides insights into operational metrics, case progress, and team performance. Reports can be viewed on-screen or exported to PDF and CSV formats for offline use and court submissions.

Reports Dashboard

The Reports section (typically accessible from the Dashboard or a dedicated Reports menu) displays available report types as cards. Each card shows the report name, description, and a "Generate" button. Previously generated reports appear in a history list with download links.

7.2 Available Report Types

7.2.1 Dashboard Summary Report

Provides a snapshot of all key metrics at a point in time:

  • Total cases by status
  • Evidence items by status and type
  • Task completion rates
  • Team workload distribution
  • Overdue items and escalation alerts
  1. Navigate to Reports
  2. Select Dashboard Summary
  3. Choose the date range (default: current month)
  4. Click Generate Report
  5. View the report on screen or click Export PDF to download

7.2.2 Case Report

A detailed report for a specific case or range of cases:

  • Case details (title, description, status, priority, timeline)
  • All tasks under the case with their statuses and assignees
  • Evidence items linked to the case with chain of custody summary
  • Case history audit trail
  • Linked cases and relationships
  1. Navigate to Reports
  2. Select Case Report
  3. Enter the Case ID or select from the dropdown
  4. Optionally include task details, evidence details, and history
  5. Click Generate Report
  6. Download as PDF for court submission or internal review

7.2.3 Activity Report

Shows all actions performed within a date range, useful for audit and compliance:

  • User login/logout events
  • Case creation and status changes
  • Evidence check-in/check-out events
  • Task assignments and completions
  • System configuration changes
  1. Navigate to Reports
  2. Select Activity Report
  3. Set the date range and optionally filter by user or action type
  4. Click Generate Report
  5. Export to CSV for further analysis in Excel or other tools
💡 Tip
Activity Reports are particularly useful for quarterly compliance audits. Schedule regular exports and archive them according to your organization's retention policy.

7.3 Exporting Reports

All reports support the following export formats:

FormatUse CaseNotes
PDFCourt submissions, formal documentationIncludes formatting, charts, and branding
CSVData analysis in Excel, import to other systemsRaw data without formatting
PrintPhysical copiesOptimized layout for standard paper sizes

7.4 Scheduling Reports

Administrators can schedule recurring reports:

  1. Generate the desired report
  2. Click Schedule instead of Export
  3. Set the recurrence (daily, weekly, monthly)
  4. Configure email delivery to specified recipients
  5. Save the schedule
ℹ Note
Scheduled reports are generated during off-peak hours (typically 2:00 AM) to minimize system impact. Email delivery requires SMTP configuration in System Settings.
